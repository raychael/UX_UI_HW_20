Government Website Redesign

A redesign of the USDA (U.S. Department of Agriculture) website, focusing on a change in navigation and page layout.

The U.S. Department of Agriculture is a government run website that focuses on the needs of Americans, Farmers, and Ranchers surrounding agriculture. User testing of the current USDA website revealed that users struggle to navigate and understand the websites purpose. Our goal was to fix the navigation and layout of the website to give it a user focused design and experience.

Current USDA Website

User Research

Proto-Persona

To begin the user research process I created a proto-persona based on what I thought an average USDA user would look like.

Typical User Path

We then created a typical user path to get an idea of how this user would attempt to navigate through the current USDA website. We annotated these webpages to find potential problems the user could run into and conducted a heuristic evaluation.

Annotated Webpages

1. User is looking to find more information on wildfires and believes this is the correct link.

2. User is taken to forest service site and inputs their location.

3. User searches location.

4. User was concerned about their own property so they click on the “Firewise Fire Information”

5. User gets resources to prepare home for a wildfire.

Usability Testing

The next step was to conduct home page usability tests. My partner and I conducted 5 usability tests with 3 tasks.

Task 1. Find the Prescott National Forest Site starting from the USDA homepage.

Success Rate: 80%

Task 2 Find the 2020-2025 Dietary Guidelines for Americans.

Success Rate: 80%

Task 3. Get to the Farmers/gov log in page.

Success Rate: 60%

Heauristic Evaluation

Navigation Heauristic Evaluation

The first step in redesigning USDA.gov was to do a heuristic evaluation of all the navigation bars on the USDA home page. I found several issues with the current navigation setup through this evaluation.

1. When we were conducting our user tests many of my users has a difficult time with this navigation bar. The home menu did not consistently lead to the same location and the other tabs did not have a clear enough meaning to the users.

Our suggestion here is making sure the home button always leads to the original home page and to make the other tabs more clear. For example, instead of “topics” we can have more categories.

2. We don’t believe there is any point for this navigation to be here because it can be found at the bottom of the screen as well. The bottom of the screen will usually be where users immediately look for these types of tabs so we’d prefer to keep them there.

3. These tabs are not clear enough to the average user coming and visiting the sight. We would prefer to get rid of this section all together.

4. We don’t like that the same navigation bar can be found at the top of the screen. This should be removed.

5. This area makes a lot of sense. I would change the wording and order for some but for the most part this section is good.

Navigation Usability Tests

We then conducted 4 usability tests and discovered that users were having a lot of problems with the current Navigation of the website.

Task 1. Using the main navigation bar discover what USDA is.

Success Rate: 60%

Task 2. Using the main navigation bar, discover what USDA does about protecting public health and safety.

Success Rate: 60%

Low-Fi Prototyping

Original Sketches

Low-Fi Wireframes

From the results of the sitemap I created the first low-fi version of the new USDA navigation system. This system broke down the categories in clearer sections so that the user didn't have to sort through a long list of options everytime they needed to find something on the website.

User Testing and Outcomes

Immediately 5 tests were conducted on the new navigation system, all lasting 5 seconds each. The goal was to make sure users did not have to think about what they were doing when using the new navigation system. After my user tests I iterated the mobile and desktop versions of the website. It was discovered that most people prefer the menu tab for their navigation for mobile. The desktop version was changed to make it easy for users to read.
